### Architecture (from Codebase Analysis)
- **Frontend**: Python GTK4 (`overlay/settings_dashboard.py` - 3500+ lines)
- **Backend**: Rust tokio daemon (`daemon/src/`)
- **HID++**: Independent implementation (`daemon/src/hidpp.rs`) - NO Solaar dependency
- **Themes**: `overlay/themes.py` (Python) + `daemon/src/theme.rs` (Rust)
- **D-Bus**: `org.kde.juhradialmx` service interface
- **Device Detection**: evdev + hidraw + libinput fallback

## Dev Notes

### Critical Constraints
1. **DO NOT break existing functionality** - Everything currently works
2. **Use Claude Plugins** - Especially LSP for type checking, linting
3. **Use MCP tools** - Leverage available integrations
4. **Consider Figma** - For designing Easy-Switch/Flow/Grid layouts before coding

### Architecture Patterns to Follow
- **Settings Dashboard**: All UI in `overlay/settings_dashboard.py`
- **Theme definitions**: Python in `overlay/themes.py`, Rust in `daemon/src/theme.rs`
- **HID++ communication**: Via `daemon/src/hidpp.rs` (independent, NOT Solaar)
- **D-Bus interface**: `daemon/src/dbus.rs` for Python ↔ Rust communication
- **Config storage**: `~/.config/juhradial/config.json`

### HID++ Features Reference
| Feature ID | Name | Purpose |
|------------|------|---------|
| 0x1814 | CHANGE_HOST | Easy-Switch device slot switching |
| 0x2111 | HIRES_SCROLL | SmartShift for modern mice |
| 0x2110 | SMARTSHIFT_LEGACY | SmartShift for older mice |
| 0x19B0 | MX_MASTER_4_HAPTIC | Haptic feedback control |
| 0x1000 | BATTERY_STATUS | Battery level queries |
| 0x1001 | BATTERY_VOLTAGE | Detailed battery info |
